The for each individual (identified by the id variable), sex and age variables are fixed at baseline and does not change. The variables date and event provides information about when something happened and what that was. In our case, we have the following possible types of event. 

table(diab_data$event)


baseline    death  dropout     MACE    visit 
    3000      890     2110      783    22655

All subjects have exactly one baseline event where date equals 0, and where baseline variables are defined. The times at events of type 'death', 'MACE', and 'dropout' denote time of death, MACE, and censoring, respectively. Death and censoring can only happens once, while MACE can happen several times. We will use first time of MACE as our event of interest and consider death as a competing event in the following analyses. The times for which event is 'visit' denotes times at which some time-varying variable was updated. In this case, the time-varying variables are HbA1c, GLP1, SGLT2, and DDP4. The HbA1c column denotes (measured) HbA1c value at the given date, while the 1/0 indicators in the GLP1, SGLT2, and DDP4 columns denote on/off treatment. For instance, the subject 1 used GLP1 at time 0 (baseline), stopped using GLP1 at time \( \approx 5.16 \) and then started using GLP1 again at time \( \approx 32.15 \).

We extract the information we need into different data sets as follows:

  • Baseline data: 

    baseline_data=diab_data[date == 0, .(id, sex, age)]

  • Outcome data: 

    ## first MACE (outcome):
out_data <- diab_data[event == "MACE" & MACE == 1][, .(id, date)]

## time of death (competing event):
comp_data <- diab_data[event == "death",.(id,date)]

## last time seen with no event time (censoring time):
cens_data <- diab_data[order(id,date), .SD[.N], by = id][event%in%c("visit","dropout"),.(id,date)]

  • Time-varying data: 

    ## Get times at which HbA1c was measured for each patient:
hba1c_data <- diab_data[event%in%c("baseline","MACE","visit"),.(id, date, value=HbA1c)]

## For each treatment variable, we construct exposure time datasets:
treat_vars <- c("GLP1","SGLT2","DPP4")
treat_data_list <- lapply(treat_vars, function(tv){
  d <- diab_data[,c("id","date",tv),with = FALSE]
  setnames(d,c("id","start_exposure","value"))
  d[,end_exposure := shift(x = start_exposure,n = 1,type = "lead"),by = id]
  ## when treatment starts at the last time point in a time-series
  ## then the duration is unknown
  d <- d[!is.na(end_exposure)]
  d <- d[value != 0]
  d[,value := NULL]
})
names(treat_data_list) <- treat_vars

Initialize rtmle object and add data

[describe init process] 

x <- rtmle_init(time_grid = seq(0, 5*6, by = 6),
                name_id = "id",
                name_competing = "death",
                name_outcome = "MACE",
                name_censoring = "censoring")

Data are then added to the empty rtmle object as follows. 

x <- add_baseline_data(x,baseline_data)
x <- add_long_data(x,
                   outcome_data = out_data,
                   censored_data = cens_data,
                   competing_data = comp_data)
x <- add_long_data(x,
                   HbA1c = hba1c_data,
                   timevar_data = treat_data_list)

Above we have exemplified different ways of adding data: Information about outcome (outcome_data, censored_data, and competing_data) have dedicated names. Time-varying variables can be added as datasets supplied with user chosen arguments (e.g., HbA1c = hba1c_data) and/or as a named list to the dedicated timevar_data argument (timevar_data = treat_data_list). As shown above, datasets can be added in one go (one add_long_data) or in separate steps (several add_long_data).

The data is now stored in the rtmle object: 

names(x$long_data)

[1] "timevar_data"   "outcome_data"   "censored_data"  "competing_data"

names(x$long_data$timevar_data)

[1] "GLP1"  "SGLT2" "DPP4"  "HbA1c"

* Aim

This vignette illustrates the main workflow for a longitudinal causal
inference analysis with =rtmle= using the simulated data set
=simulated_cohort=.  The data contain simulated event-history register data with
baseline covariates, repeated visits, treatment assignments, intermediate
events, and absorbing events.

The examples below estimate the absolute risks of =stroke= in the
presense of the competing risk =death without stroke= under different
static treatment protocols.

** Convert the event history to register format

The =rtmle= long-data workflow expects data in a similar format as
they occur naturally in electronic health records where baseline data
and data from the person-specific follow-up are memory efficiently
saved as separate objects. We use the function
=rtmle::register_format()= to transform the above data into a
corresponding list of objects.

#+BEGIN_SRC R
# Input: simulated_cohort, an object of class "simulated_cohort".
# Output: ld, a list with baseline_data and timevar_data components.
ld <- register_format(simulated_cohort)

names(ld)
names(ld$timevar_data)
NROW(ld$baseline_data)
#+END_SRC

The resulting list contains one baseline table and one named =timevar_data=
entry for each visit variable and event variable.  In this example:

- =ld$baseline_data= contains one row per subject with baseline covariates.
- =ld$timevar_data$stroke= contains outcome event times.
- =ld$timevar_data$death= contains competing event times.
- =ld$timevar_data$dropout= contains censoring times.
- =ld$timevar_data[c("bleeding", "changeSBP", "A", "B")]= contains
  time-varying covariates and treatments.

* Initialize the =rtmle= analysis object

The call to =rtmle_init()= records the analysis time grid and the
variable names that identify subjects, time, outcome, competing event,
and censoring along with the following 3 tuning parameters:

- =minority_threshold=: (default=8) When fitting propensity score,
  censoring and (the first) outcome regression model of the sequence,
  regression modelling is skipped and the mean outcome is predicted
  when the total number of subjects with a one (or with a zero) is
  below this threshold.
- =weight_truncation=: (default =c(0,1)=) Inverse probability weights are truncated at these bounderies.
- =prediction_range=: (default =c(0.0001,0.9999)=) Before the tmle update step of the sequential regression algorithm, the predicted outcome values are truncated in this way to allow logit transformation.

The supplied time grid =seq(0, 20, 4)= defines a discrete time scale
with 5 intervals for the LTMLE analysis.

#+BEGIN_SRC R
# Input: variable names and a discrete analysis time grid.
# Output: x, an empty "rtmle" object with names, time grid, and tuning settings.
x <- rtmle_init(
    time_grid = seq(0, 20, 4),
    name_id = "id",
    name_outcome = "stroke",
    name_competing = "death",
    name_censoring = "dropout",
    censored_label = "censored"
)

x
#+END_SRC

** Add timevarying covariate information and outcome data (including date of competing risks and censoring)

=add_long_data()= attaches outcome, competing-risk, censoring,
treatment, and time-varying covariate histories as measured on the
original continuous time scale to the initialized object. The outcome
data must have at most one row per subject, so the example keeps the
first =stroke= event for each subject.

#+BEGIN_SRC R
# Input: x from rtmle_init() and the long-format event/covariate tables in ld.
# Output: x with long_data entries for stroke, death, dropout, bleeding,
# changeSBP, A, and B.
x <- add_long_data(
    x,
    outcome_data = ld$timevar_data$stroke[!duplicated(id)],
    censored_data = ld$timevar_data$dropout,
    competing_data = ld$timevar_data$death,
    timevar_data = ld$timevar_data[c("bleeding", "changeSBP", "A", "B")]
)

* Discretize the longitudinal data onto the discrete time scale

time-varying variables.

# Output: x$data contains wide outcome and time-varying data on the time grid.
x <- long_to_wide(x, start_followup_date = 0)

names(x$data)

* Define treatment protocols

The function =protocol()= adds an intervention to the object.  A
simple example is a static regime which sets treatment =A= to 1 at
every intervention node.  The intervention values are factors so that
=rtmle= knows the two treatment options. It is recommended that one
checks the intervention table after adding the intervention:

#+BEGIN_SRC R
# Input: x with prepared_data and intervention nodes.
# Output: x$protocols$Always_A with an intervention table and adherence matrix.
x <- protocol(
    x,
    name = "Always_A",
    intervention = data.frame(
        node = x$intervention_nodes,
        A = factor("1", levels = c("0", "1"))
    ),
    verbose = FALSE
)

x$protocols$Always_A$intervention_table

The protocol function can add more complex interventions that depend
on multiple variables and apply user-written intervention
functions. Here is an example of a two-factor regime which states that
people should start treatment 'A', continue treatment 'A' for 20
months, that is until node 4 on the discrete time grid, but are not
allowed to take treatment 'B' in the first 10 months:

#+BEGIN_SRC R
# Input: x with prepared_data and intervention nodes.
# Output: x$protocols$Always_A_not_B_in_the_beginning with an intervention table and adherence matrix.
x <- protocol(
    x,
    name = "Always_A_not_B_in_the_beginning",
    intervention = data.frame(
        time = x$intervention_nodes,
      A = factor(rep("1",5), levels = c("0", "1")),
      B = factor(c("0","0",NA,NA,NA), levels = c("0", "1"))

    verbose = FALSE
)

* Specify nuisance-parameter model formulas

=model_formula()= creates formulas for the outcome, censoring, and
treatment models required by the sequential regression and TMLE update
step. The formulas are derived from the prepared data, the time grid,
and the treatment variables intervened on by the protocols. In
addition to selecting variables that should be adjusted for, it is
often useful to consider which variables should not be adjusted
for. E.g., if =A_0= is equal to =(1-B_0)= then it makes little sense
to have both variables in the adjustment set. To show how this works we here remove 'B_0' from all models
and all bleeding variables from all censoring models:

#+BEGIN_SRC R
# Input: x with prepared_data, protocols, and target definition.
# Output: x$models, a nested list of formulas indexed by time and model node.
x <- model_formula(x, verbose = FALSE,exclusion_rules = list("*" = "B_0","dropout" = "bleeding"))

names(x$models)

* Run the longitudinal TMLE

=run_rtmle()= fits the nuisance-parameter models, performs the sequential
regression and the TMLE update step, and stores estimates in =x$estimate=.  The example
uses =learn_glm= to fit all models. Other implemented learners are =learn_glmnet=, =learn_ranger= and =learn_xgboost=
as well as super-learner
strategies (discrete and ensemble super learners) that can be specified through the =learner= argument.

#+BEGIN_SRC R
# Input: x with prepared_data, protocols, targets, and model formulas.
# Output: x with fitted nuisance models, influence-curve components, and
# main-analysis estimates for the requested time horizons.
analysis_horizons <- 1:2

x <- run_rtmle(
    x,
    learner = "learn_glm",
    time_horizon = analysis_horizons,
    verbose = FALSE
)

x

* Inspect the fit, check adherence and inverse-probability weights

The fitted object also contains protocol diagnostics.  The adherence matrices
record whether each subject followed the protocol through each intervention
node.  The cumulative intervention probabilities are used to construct the
inverse-probability weights in the TMLE update.
There are useful plot functions:

- =rtmle::plot.rtmle()=
- =rtmle::plot_model_coefficients()=
- =rtmle::plot_adherence()=
- =rtmle::plot_ipw()=

* Complete script

For reference, the complete workflow is the sequence of the state-changing
commands above:

#+BEGIN_SRC R
library(rtmle)
library(data.table)

data(simulated_cohort)
ld <- register_format(simulated_cohort)

x <- rtmle_init(
    time_grid = seq(0, 20, 4),
    name_id = "id",
    name_outcome = "stroke",
    name_competing = "death",
    name_censoring = "dropout",
    censored_label = "censored"
)

x <- add_long_data(
    x,
    outcome_data = ld$timevar_data$stroke[!duplicated(id)],
    censored_data = ld$timevar_data$dropout,
    competing_data = ld$timevar_data$death,
    timevar_data = ld$timevar_data[c("bleeding", "changeSBP", "A", "B")]
)

x <- add_baseline_data(x, data = ld$baseline_data)
x <- long_to_wide(x, start_followup_date = 0)
x <- prepare_rtmle_data(x)

x <- protocol(
    x,
    name = "Always_A",
    intervention = data.frame(
        time = x$intervention_nodes,
        A = factor("1", levels = c("0", "1"))
    ),
    verbose = FALSE
)

x <- protocol(
    x,
    name = "Always_A_not_B_in_the_beginning",
    intervention = data.frame(
        time = x$intervention_nodes,
      A = factor(rep("1",5), levels = c("0", "1")),
      B = factor(c("0","0",NA,NA,NA), levels = c("0", "1"))
    ),
    verbose = FALSE
)

x <- target(
    x,
    name = "Stroke_risk",
    protocols = c("Always_A", "Always_A_not_B_in_the_beginning"),
    estimator = "tmle"
)

x <- model_formula(x, verbose = FALSE)
x <- run_rtmle(x, learner = "learn_glm", time_horizon = 1:2, verbose = FALSE)

x
#+END_SRC
